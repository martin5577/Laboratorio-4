# Laboratorio-4
Aplicación de diseño inclusivo y accesibilidad a Portafolio Web Responsivo
## Análisis de referencia
Como parte del proceso, se realizó una revisión de la página web de la Universidad UISEK de Ecuador, identificando oportunidades de mejora relacionadas con accesibilidad, estructura semántica y experiencia de usuario. A partir de este análisis, se aplicaron buenas prácticas de accesibilidad en el desarrollo del presente portafolio web.

## Herramientas utilizadas
- HTML5 semántico
- CSS3
- Bootstrap (tema personalizado generado con bootstrap.build)
- GitHub para control de versiones

## Mejoras de accesibilidad implementadas

### 1. Estructura semántica correcta
Se utilizaron etiquetas semánticas de HTML5 como:
- `<header>`
- `<nav>`
- `<main>`
- `<section>`
- `<article>`
- `<footer>`

Esto mejora la comprensión del contenido por parte de lectores de pantalla y tecnologías de asistencia.

### 2. Jerarquía adecuada de títulos
Se definió una jerarquía clara y coherente de encabezados (`h1`, `h2`, `h3`), facilitando la navegación y comprensión de la estructura del sitio.

### 3. Contraste adecuado entre texto y fondo
El tema de interfaz fue generado con colores que cumplen con el contraste mínimo recomendado por las WCAG (relación 4.5:1), mejorando la legibilidad para personas con baja visión o daltonismo.

### 4. Imágenes con texto alternativo descriptivo
Todas las imágenes incluyen el atributo `alt` con descripciones claras y significativas, permitiendo que los usuarios de lectores de pantalla comprendan su contenido.

### 5. Enlaces y botones con descripciones claras
Se evitaron textos genéricos como “haz clic aquí”, utilizando en su lugar descripciones claras que indican la acción o destino del enlace o botón.

### 6. Formularios accesibles
Los formularios incluyen:
- Etiquetas `<label>` asociadas correctamente a cada campo.
- Campos claramente identificados.
- Botones con texto descriptivo.

Esto mejora la accesibilidad para lectores de pantalla y usuarios con discapacidades cognitivas.

### 7. Navegación mediante teclado
El sitio es completamente navegable utilizando el teclado (tecla Tab), incluyendo:
- Enlace para saltar al contenido principal.
- Foco visible en enlaces y botones.

### 8. Mejoras de diseño inclusivo
Se aplicaron las siguientes mejoras específicas:
- Tamaños de fuente ajustables y legibles.
- Foco visible para navegación por teclado.
- Diseño responsivo adaptable a distintos dispositivos y tamaños de pantalla.

## Cumplimiento de pautas WCAG
Las mejoras implementadas se alinean principalmente con:
- WCAG 2.1 – Principio Perceptible
- WCAG 2.1 – Principio Operable
- WCAG 2.1 – Principio Comprensible
